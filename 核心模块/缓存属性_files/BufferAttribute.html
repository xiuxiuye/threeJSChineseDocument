缓存属性（BufferAttribute）

这个类保存了和 缓存几何模型(BufferGeometry) 关联的属性数据。该类用来保存内置属性比如顶点位置、法相量和颜色等，也可以被用于保存自定义属性。

构造器（Constructor）

BufferAttribute( array, itemSize, normalized )

从相关缓存中的数据来实例化这个属性。 itemSize 给定缓存中和1个顶点关联的数组中数据的个数。normalized 表示如何将相关的缓冲区中的数据映射到GLSL着色器中代码的值。

属性（Properties）

# .array

用来保存和这个属性关联的数据。应该有 itemSize * numVertices 个元素，这里 numVertices 是相关联的缓存几何模型(BufferGeometry)的顶点个数。array 可以是 UInt8Array, Int8Array, UInt16Array, Int16Array, 或 Float32Array的实例。

# .itemSize

记录一个顶点和多少数组项关联。比如，如果这个属性用来保存3维向量（比如坐标、法向量或颜色），那么 itemSize 应该是3。

# .length

数组元素的总数目。

# .needsUpdate

标识该属性是否被改变并应该重新提交给GPU。当你修改数组数据时，应设置该属性为true。

# .normalized

标识如何将关联的缓冲区中的数据映射到GLSL代码中的值，也就是是否正规化。比如，如果 array 是UInt16Array的一个实例，
并且 normalized 为true，那么数组数据中的 0 - +65535 取值将被映射为GLSL属性中的 0.0f - +1.0f。
而一个Int16Array（有符号的）将把 -32767 - +32767 映射到 -1.0f - +1.0f。
如果 normalized 为 false，那么这些值将被转换成相等的浮点数，也就是 32767 会映射成 32767.0f。

# .version

版本号，但needsupdate属性设置为true时递增。

方法（Methods）

# .copyAt ( index1, attribute, index2 )

拷贝数组中itemSize个数值，从index2处的顶点（vertex）到index1处的顶点（vertex）。

# .set ( value )

从传递的数组中取值来设置关联数组。

# .setX ( index, x )

设置数组位于index * itemSize的值为 x。

# .setY ( index, y )

设置数组位于index * itemSize + 1的值为 y。

# .setZ ( index, z )

设置数组位于index * itemSize + 2的值为 z。

# .setXY ( index, x, y )

设置数组位于index * itemSize的值为 x，并且 设置数组位于index * itemSize + 1的值为 y。

# .setXYZ ( index, x, y, z )

设置数组位于index * itemSize的值为 x，且 设置数组位于index * itemSize + 1的值为 y，且 设置数组位于index * itemSize + 2的值为 z。

# .setXYZW ( index, x, y, z, w )

设置数组位于index * itemSize的值为 x，且 设置数组位于index * itemSize + 1的值为 y，且 设置数组位于index * itemSize + 2的值为 z，且 设置数组位于index * itemSize + 3的值为 w。

# .clone ()

拷贝属性。
